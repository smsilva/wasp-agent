# Async watcher

- `wasp/watcher.py` — `watch_platform(name, chat_id, token)`: polls Platform CR via `kubernetes.client.CustomObjectsApi`, 10s/poll, 10min timeout, notifies via `httpx.AsyncClient` POST directly to the Telegram API.
- Spawn: agno calls synchronous tools in a thread executor (no active event loop), so `asyncio.get_running_loop()` and `get_event_loop()` fail. Use `threading.Thread(target=asyncio.run, args=(watch_platform(...),), daemon=True).start()` — creates its own event loop in the daemon thread.
- k8s API 404 when fetching the freshly committed Platform CR is expected (ArgoCD hasn't synced yet). Treat as "not created yet": `sleep(POLL_INTERVAL)` and `continue`, do not notify error or return. Only notify timeout after `WATCH_TIMEOUT_SECONDS`.
- The watcher is fire-and-forget in a daemon thread — exceptions are silently swallowed. Wrap internal logic in `_watch_platform_inner` called via `try/except Exception: log.exception(...)` in the public wrapper.
- Dev-only: file edits during uvicorn `--reload` kill async tasks in flight (watchfiles restarts the process). Not an issue in production without `--reload`.
- `run_context=None` in `provision_platform_instance` — agno injects the execution context; `extract_chat_id` parses `session_id = "tg:{entity}:{chat_id}"`. If not a Telegram session or `TELEGRAM_TOKEN` is not set, watcher is silently not spawned.
- `RunContext` is in `agno.run.base` (agno 2.6.5).
- `pytest-asyncio` version 1.3.0 uses strict mode by default — add `asyncio_mode = "auto"` in `[tool.pytest.ini_options]` to avoid `@pytest.mark.asyncio` on every test.
- kubernetes mocks in conftest: `ConfigException` and `ApiException` are MagicMock — cannot be used in `raise`/`except`. For tests that need these exceptions, create real `FakeConfigException(Exception)` / `FakeApiException(Exception)` and `monkeypatch.setattr(w.config, "ConfigException", FakeConfigException)`.
- To test watcher spawn, mock `threading.Thread` via `patch("tools.provision.threading.Thread", mock_thread_cls)` and assert `mock_thread_cls.assert_called_once()` + `mock_thread.start.assert_called_once()`. Do not try to execute the real coroutine in tests.
- When mocking `time.monotonic` with `iter([...])` in async tests: event loop teardown calls `time.monotonic()` extra times, causing `StopIteration`. Use `itertools.chain([...], repeat(last_value))` for a never-exhausting iterator.
- `coroutine 'watch_platform' was never awaited` warning in tests that mock `loop.create_task`: harmless — the mock doesn't execute the coroutine, which is collected by the GC. Do not suppress; it is an expected mock setup artifact.
- `notify_telegram` uses `httpx.AsyncClient` directly — does not reuse the agno/Telegram client (no access). Overhead is acceptable: one connection per notification.
